# HUD Components

EclipsePlus provides on-screen HUD components that display real-time progression data.

## Overview

HUD components are small, unobtrusive progress bars that show:
- Current progress values
- Level/rank indicators
- Visual feedback for gains

---

## Available Components

### Experience Bar

Displays player level and XP progress:
- Current level number
- XP progress bar
- Percentage to next level

### Expertise Bar

Shows weapon mastery for equipped weapon:
- Weapon type icon
- Expertise level
- Progress to next level

### Familiar Bar

Active familiar status:
- Familiar name
- Current level
- Health indicator (when in combat)

### Legacy Bar

Blood legacy progress for current blood type:
- Blood type icon
- Legacy level
- Progress to next level

### Quest Tracker

Active quest objectives:
- Quest name
- Progress indicator
- Target information

---

## Layout Editor

Press **F9** to enter Layout Mode:

### Features
- **Drag positioning** — Move HUD elements anywhere
- **Persistent saves** — Layout saved per-character
- **Reset option** — Return to default positions

### How to Use

1. Press F9 to toggle Layout Mode
2. Hover over HUD elements to highlight
3. Click and drag to reposition
4. Press F9 again to save and exit

<Callout type="tip">
  HUD positions are saved locally and persist across sessions.
</Callout>

---

## Visibility

HUD components automatically show/hide based on:
- **System enabled** — Only shows if server has the system active
- **Data available** — Waits for server sync before displaying
- **Context** — Some bars hide when not relevant

---

## Implementation

### Component Architecture

Each HUD component follows a base pattern:

```
Services/HUD/
├── Base/
│   └── HudComponentBase.cs    # Shared behavior
├── ExperienceBar/
│   └── ExperienceBarComponent.cs
├── ExpertiseBar/
│   └── ExpertiseBarComponent.cs
├── FamiliarBar/
│   └── FamiliarBarComponent.cs
├── LegacyBar/
│   └── LegacyBarComponent.cs
├── QuestTracker/
│   └── QuestTrackerComponent.cs
└── Shared/
    └── HudData.cs             # Sprite allowlists
```

### Data Binding

HUD components read from `DataService`:
- Subscribe to data change events
- Update visuals when data changes
- Handle missing/null data gracefully

---

## Customization

### For Server Admins

HUD component visibility is tied to server config:
- Disable a system → HUD component won't appear
- Sync frequency controlled by `Eclipsed` setting

### For Developers

To add a new HUD component:

1. Create component class extending `HudComponentBase`
2. Register in `CanvasService`
3. Add data handling in `DataService`
4. Add sprites to `HudData.cs` allowlist

See [Contributing](/contributing) for development setup.
